# Repository Instructions

Luôn dùng file này như cổng vào ngữ cảnh của repo.

Trước khi sửa code hoặc tài liệu trong repo này:

1. Đọc `docs/rule.md` để nắm quy tắc AI/coding.
2. Đọc phần liên quan trong `docs/dev.md` để nắm bối cảnh dự án, cấu trúc, lệnh vận hành và workflow.
3. Đọc source/config gần nhất với yêu cầu trước khi chỉnh sửa.

Nếu task rất nhỏ, chỉ cần đọc phần liên quan trong `docs/dev.md` để tiết kiệm token, nhưng vẫn phải tuân thủ `docs/rule.md`.
